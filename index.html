VibeVoice-Realtime
Real-time voice AI • Microsoft 0.5B model
GPU Required Google Colab T4 GPU

How to launch

OmniVoice-TTS
Voice cloning • local app + Colab relay • ~4 GB model
Voice Clone Local App Google Colab T4 GPU

Zero-shot voice cloning from 3-second samples. Cross-lingual TTS (English voice → Spanish speech), emotional tags [laughter] [sigh] [gasps], advanced inference controls, and experimental Voice Design dropdowns.

Local app (Windows & Linux)

  1. Windows: run apps\omnivoice\install.bat once, then start.bat.
  2. Linux: run apps/omnivoice/install.sh once, then ./start.sh.
  3. Open http://localhost:8765 — default backend keeps the model loaded in memory (no per-chunk reload).
  4. For GPU speed, flip backend to Remote — Colab relay and paste the URL from Step 5 of the Colab notebook.

Colab (Gradio UI or relay for local app)

  1. Click "Open in Colab" → set runtime to T4 GPU.
  2. Run Step 1 → Step 2 (install).
  3. Run Step 3 for the full Gradio UI, or Step 5 to serve as a relay for the local app. Do not run both.

Fish-Speech
Text-to-speech • fishaudio / v2.0.0
TTS Self-hosted API + WebUI

High-quality, multi-language text-to-speech with voice cloning. Runs locally or on any GPU server. Includes a REST API and a Gradio web UI.

How to launch

  1. Initialize submodule: git submodule update --init --recursive
  2. Install deps: cd fish-speech && pip install -e .[stable]
  3. Download model weights from Hugging Face (see fish-speech README).
  4. Start WebUI: python -m tools.run_webui --listen
  5. Open http://localhost:7860 in your browser.

Host on a GPU Cloud
For 10 – 50 concurrent users
Paid GPU Scalable

Standard VPS (CPU-only) is too slow for real-time voice AI. You need a GPU instance. Options range from cheap spot GPUs to full managed servers.

Recommended platforms

  1. RunPod.io — cheapest on-demand GPUs (RTX 3090 ~$0.34/hr). Deploy a pod, run your server, expose a port.
  2. Vast.ai — peer-to-peer GPU marketplace, even cheaper. Good for Fish-Speech.
  3. Lambda Labs — reliable GPU cloud, good uptime for production.
  4. Google Colab Pro+ — easiest if you already use Colab. Background execution keeps sessions alive.
  5. Your VPS + queue — see the "Queue Architecture" card below.
⚡
Queue Architecture
Serve 10–50 users from one GPU
Nginx Redis Workers

One GPU handles requests one at a time very fast. A request queue lets your cheap VPS accept all connections while the GPU cloud does the heavy lifting.

Architecture

  1. VPS (nginx) → accepts HTTP from all users, cheap ($5/mo).
  2. Redis queue on VPS → holds pending jobs.
  3. GPU worker (RunPod/Vast) → pulls from queue, runs inference, returns result.
  4. Users get a job ID and poll for results — no dropped connections.
  5. Add more GPU workers horizontally to scale past 50 users.
Capacity: 10–50 users on a single GPU worker

Buzz
Audio transcription • OpenAI Whisper • Offline
Runs Locally Windows / Mac / Linux Whisper

Transcribe any audio or microphone input to text using OpenAI Whisper — fully offline, no API key needed. Supports 90+ languages. Perfect for transcribing client calls, meetings, or creating subtitles.

How to install

  1. Go to the GitHub releases page and download the installer for your OS.
  2. Run the installer — no Python or GPU required for CPU mode.
  3. Open Buzz → choose a Whisper model size (Small = fast, Large = accurate).
  4. Record live audio or load an audio/video file to transcribe.
  5. Export transcript as .txt, .srt (subtitles), or .vtt.

Hosting at a Glance

🖥️ CPU VPS Only

  • Works for fish-speech at low quality / slow speed
  • NOT suitable for real-time voice conversion
  • Good for: frontend, API gateway, queue server
  • Cost: ~$5–$20/mo
Users: 1–5 (TTS only, slow)

⚡ GPU Cloud (RunPod/Vast)

  • Full speed for both apps
  • Spin up/down on demand — pay per hour
  • RTX 3090: ~$0.30–$0.50/hr
  • One GPU handles ~10–20 concurrent requests with queuing
Users: 10–30 per GPU

🏢 Managed GPU (Lambda/AWS)

  • Higher reliability and uptime SLAs
  • Auto-scaling with load balancer
  • More expensive but production-ready
  • A10G/A100 handles heavy concurrent load
Users: 30–50+ scalable

🎓 Google Colab Pro+

  • Easiest to set up if you use Colab already
  • Background execution: session stays alive
  • Cloudflare Tunnel gives public URL for free
  • ~$50/mo, limited to one session at a time
Users: 5–15 (single session)
